# CONTEXT

## 이 모듈이 하는 일
- 운영/설계/변경 맥락 문서를 구조화해 코드 이해 비용을 낮춘다.

## 파일 구조와 역할
- `docs/README.md`: 문서 인덱스
- `docs/domains/*`: 도메인별 정책/흐름/운영 기준
- `docs/context/changes/*`: 날짜/도메인별 변경 맥락 기록

## 핵심 설계 결정
- `README`는 개발환경 정보만 유지하고, 도메인 지식은 `docs`로 분리한다.
- 변경 맥락 문서는 `docs/context/changes/{domain}-{YYYYMMDD}.md` 패턴으로 관리한다.

## 사용 패턴
- 기능 변경 시 해당 도메인 문서와 변경 맥락 문서를 함께 갱신한다.
- 코드 리뷰/온보딩 시 `docs/README.md`를 진입점으로 사용한다.

## 확장 시 고려사항
- 도메인 문서가 과도하게 커지면 하위 주제 파일로 분할한다.
- 문서 제목/경로 네이밍은 팀 컨벤션(소문자-kebab-case)을 유지한다.
